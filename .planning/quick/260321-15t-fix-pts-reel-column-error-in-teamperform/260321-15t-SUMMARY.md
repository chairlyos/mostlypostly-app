# Quick Task 260321-15t: Fix pts_reel Column Error in Team Performance

**One-liner:** Added missing `pts_reel` column via migration 053 and sorted Performance settings point categories by value descending.

## Summary

The Performance page (`/manager/performance`) crashed with `SqliteError: no such column: pts_reel` when saving settings. Root cause: migration 025 created `gamification_settings` with 7 point columns but omitted `pts_reel`, even though `DEFAULT_POINTS` in `gamification.js` has always included `reel: 20`. The POST settings handler built its UPDATE SET clause from `Object.keys(DEFAULT_POINTS)`, so `pts_reel` was always included in the query — against a table column that didn't exist.

Additionally, the point values card displayed categories in JavaScript object insertion order (arbitrary), making it harder for managers to orient. The fix sorts by current value descending so the highest-value post types (Reel 20pts, Before & After 15pts) appear first.
