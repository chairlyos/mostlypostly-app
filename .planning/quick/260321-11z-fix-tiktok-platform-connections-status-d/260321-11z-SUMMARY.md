# Phase quick Plan 260321-11z: Fix TikTok Platform Connections Status Display Summary

**One-liner:** TikTok row now shows Connected/Not connected status badge matching FB/IG pattern, and its daily cap input moved to the Platform Daily Caps card as a third column.

## What Was Built

Two targeted UI fixes in `src/routes/schedulerConfig.js`:

1. **TikTok Platform Connections row** — replaced the inline `<input type="number">` with a status pill. The pill shows green "Connected" when `salon.tiktok_account_id && salon.tiktok_refresh_token` are both truthy, gray "Not connected" otherwise. Subtitle now shows `@username` when connected or guidance text when not, matching the FB/IG row pattern exactly.

2. **Platform Daily Caps card** — added TikTok as a third column (`sm:grid-cols-3`). Input has `id="tkCapInput"`, `name="tiktok_daily_max"`, and `oninput="updatePace()"` wired in. Recommended range shows "Rec: 1–3". Section comment updated from "FB + IG only" to "FB + IG + TikTok".

The `updatePace()` JS function was reviewed — it calculates effective daily from `min(ig, fb)` since one post publishes to both. TikTok is a separate independent platform so it was intentionally not added to the min() calculation. The `tkCapInput` id is registered for future use.
