In the script block at the bottom of the GET / handler's body template, add the following client-side JS after the existing SortableJS/day-panel code, inside the same IIFE:

**A. localStorage keys and defaults:**
```javascript
var LS_VIEW = 'calendar_view';
var LS_FILTERS = 'calendar_filters';
var LS_CARD = 'calendar_card_settings';

var DEFAULT_FILTERS = {
  types: { standard_post: true, before_after: true, promotion: true, availability: true, celebration: true, reel: true, vendor: true },
  statuses: { manager_pending: true, manager_approved: true, published: true, failed: true }
};
var DEFAULT_CARD = { showStylist: true, showPlatforms: true, showTime: true, showCaption: true };

function loadJSON(key, fallback) {
  try { return JSON.parse(localStorage.getItem(key)) || fallback; }
  catch(e) { return fallback; }
}
function saveJSON(key, val) { localStorage.setItem(key, JSON.stringify(val)); }
```

**B. View toggle logic:**
- Read `activeView` from `localStorage.getItem(LS_VIEW)` or default `'month'`
- `setActiveViewBtn(view)`: loop `.view-btn` buttons. Active button gets `bg-[#3B72B9] text-white`. Inactive gets `bg-white text-[#2B2D35] hover:bg-[#F8FAFC]`.
- `switchView(view)`: saves to localStorage, sets active button, hides nav-arrows for agenda, and:
  - If `view === 'month'`: reload page via `window.location.href = '/manager/calendar' + window.location.search`
  - If `view === 'week'` or `'agenda'`: fetch `/manager/calendar/${view}` (with query string for week), set response HTML into `#calendar-view-body` via `.innerHTML`, then call `applyFilters()` and `applyCardSettings()`
- Attach click handler to each `.view-btn`
- On page load: call `setActiveViewBtn(activeView)`, if not month call `switchView(activeView)`, if agenda hide `#nav-arrows`

**C. Filter logic:**
- Load filters from localStorage: `var filters = loadJSON(LS_FILTERS, DEFAULT_FILTERS)`
- `applyFilters()`: loop `.calendar-post-card` and `.agenda-post-card` elements. Read `dataset.postType` and `dataset.status`. Toggle `hidden` class based on `filters.types[type] !== false` AND `filters.statuses[status] !== false`. Then call `checkEmptyState()`.
- `renderFilterChips()`: loop `.filter-chip` buttons. Read `dataset.filterType` or `dataset.filterStatus`. Toggle `opacity-30` class based on filter state.
- Click handler on each `.filter-chip`: toggle the matching filter key, save to localStorage, re-render chips, re-apply filters.
- Call `renderFilterChips()` and `applyFilters()` on load.

**D. Card settings logic:**
- Load card settings: `var cardSettings = loadJSON(LS_CARD, DEFAULT_CARD)`
- `applyCardSettings()`: map each setting key to a CSS class: `showStylist` to `.card-field-stylist`, `showPlatforms` to `.card-field-platforms`, `showTime` to `.card-field-time`, `showCaption` to `.card-field-caption`. Toggle `display: none` on elements matching each class based on setting value.
- Gear dropdown open/close: click on `#card-settings-btn` toggles `hidden` on `#card-settings-dropdown`. Click anywhere outside `#card-settings-wrapper` hides the dropdown.
- Each `.card-setting-toggle` checkbox: set `checked` from cardSettings on load. On `change`, update cardSettings, save to localStorage, call applyCardSettings().
- Call `applyCardSettings()` on load.

**E. Empty filter state:**
- `checkEmptyState()`: count all `.calendar-post-card, .agenda-post-card` in `#calendar-view-body`. If all have `hidden` class (and count > 0), show a `<p id="filter-empty-msg">No posts match the current filters.</p>` element. If any visible, hide it. Create the element dynamically if it doesn't exist.
