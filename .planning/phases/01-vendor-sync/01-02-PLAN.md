**Main export — `runVendorSync(vendorName?)`:**
- If vendorName provided, find that config only; otherwise loop all VENDOR_CONFIGS
- For each config: check syncInProgress guard, set it, try/finally delete it
- Call `syncVendor(config)` which does the full pipeline
- On success: update vendor_brands SET last_sync_at, last_sync_count, last_sync_error = NULL
- On error: update vendor_brands SET last_sync_error = err.message.slice(0, 500)
- Return `{ vendor, imported, skipped, error }` result object

**syncVendor(config) pipeline steps:**

Step 1 — Login + PDF download (`downloadPortalPdf(config)`):
- Read credentials from `process.env[config.credentialEnvVars.userEnv]` and `.passEnv`
- If missing, throw Error with descriptive message
- `const browser = await getBrowser()` — MUST use singleton
- `const page = await browser.newPage()`
- try/finally with `await page.close()` in finally
- Navigate to config.portalUrl with waitUntil 'domcontentloaded', timeout 30000
- Fill login form: `await page.type(config.loginSelectors.userSelector, user)`
- `await page.type(config.loginSelectors.passSelector, pass)`
- `await page.click(config.loginSelectors.submitSelector)`
- `await page.waitForNavigation({ waitUntil: 'domcontentloaded', timeout: 30000 })`
- Build search keyword: replace `{MONTH}` with current month name, `{YEAR}` with current year
- Type search keyword into search input, submit
- Wait for results, click first matching card (config.pdfDownloadSelectors.cardSelector)
- Set up CDP download: `const cdpSession = await page.createCDPSession()`
- `const downloadDir = path.join(os.tmpdir(), 'vendor-pdf-' + Date.now())`
- `fs.mkdirSync(downloadDir, { recursive: true })`
- `await cdpSession.send('Browser.setDownloadBehavior', { behavior: 'allow', downloadPath: downloadDir, eventsEnabled: true })`
- Set up download completion promise with 120s timeout + `Browser.downloadProgress` event listener
- Also implement file-system polling fallback: if CDP event doesn't fire within 10s, switch to polling downloadDir every 500ms for a .pdf file not ending in .crdownload
- Click download button (config.pdfDownloadSelectors.downloadButtonSelector)
- Wait for download completion
- Return path to downloaded PDF file

Step 2 — Parse PDF (`parseCampaignPdf(pdfPath, config)`):
- Read file to Uint8Array: `new Uint8Array(fs.readFileSync(pdfPath))`
- `const doc = await pdfjsLib.getDocument({ data, standardFontDataUrl: path.join(__dirname, '../../node_modules/pdfjs-dist/standard_fonts/') }).promise`
- Loop pages from 2 to doc.numPages (skip page 1 = cover)
- For each page: `const textContent = await page.getTextContent()`
- `const annotations = await page.getAnnotations()`
- Extract text items, sort by Y position descending (item.transform[5]) to reconstruct reading order
- Group items into lines by Y-threshold (items within 3 units of Y = same line)
- Join items per line to get `lines[]` array
- Extract links: `annotations.filter(a => a.subtype === 'Link' && a.url).map(a => a.url)`
- Also scan text lines for URLs via regex `/(https?:\/\/[^\s]+)/`
- Call `extractCampaignFromPage(lines, allUrls, pageNum, config)` to get structured campaign

**extractCampaignFromPage(lines, urls, pageNum, config):**
- release_date: find first line matching config.pdfParserHints.dateRegex
- campaign_name: first short line (< 60 chars) in top 5 lines that is NOT the date
- caption_body: the longest multi-line text block, or the block containing `[SALON NAME]`; join adjacent lines until a hashtag line is hit
- product_hashtag: lines starting with `#` near bottom; join them; store as single string
- imageUrl: first URL from annotations or text that looks like an image resource (not a login page URL)
- Return `{ release_date, campaign_name, caption_body, product_hashtag, imageUrl, pageNum }`

Step 3 — Download images (`downloadCampaignImage(imageUrl, config, page)`):
- Create dir: `const vendorDir = path.join(UPLOADS_DIR, 'vendor', config.imageSubdir)` and `fs.mkdirSync(vendorDir, { recursive: true })`
- filename: `campaign-${Date.now()}-${crypto.randomUUID().slice(0,8)}${ext}` where ext comes from URL or defaults to '.jpg'
- If config.imageDownloadStrategy === 'auto':
  - Try HEAD request first: `const head = await fetch(imageUrl, { method: 'HEAD' })`
  - If response.ok and content-type starts with 'image/', do direct fetch and write to file
  - If not (401, 302, text/html), use Puppeteer page-click: navigate to imageUrl in the same authenticated page, find download button, use CDP download
- If 'direct': always fetch
- If 'page-click': always Puppeteer
- Return the relative path suitable for photo_url: `/uploads/vendor/${config.imageSubdir}/${filename}`

Step 4 — Dedup + Insert (`insertCampaigns(campaigns, config)`):
- For each campaign in the parsed array:
  - Normalize: `campaign_name.trim()`, `release_date` to ISO-ish format
  - `const id = crypto.randomUUID()`
  - Use `db.prepare('INSERT OR IGNORE INTO vendor_campaigns ...').run({...})` — synchronous, NO await
  - Fields: id, vendor_name (config.vendorName), campaign_name, release_date, caption_body (verbatim with [SALON NAME]), product_hashtag, photo_url (from image download), expires_at (null or 60 days from release_date), frequency_cap (4), active (1), source ('pdf_sync'), created_at (new Date().toISOString())
  - Track inserted count via `result.changes`
- Return `{ imported: insertedCount, skipped: totalParsed - insertedCount }`

**Cleanup:** Delete temp PDF directory after parsing is complete (in finally block).

**Logging:** Use `console.log('[VendorSync]', ...)` for info, `console.error('[VendorSync]', ...)` for errors. Log each step: login, search, download, parse page count, image download, insert count, skip count.

**Error handling:** Wrap the entire pipeline in try/catch. On any error, update vendor_brands.last_sync_error and re-throw. Individual image download failures should log a warning but not abort the entire sync — skip that campaign's image and continue.

**Install pdfjs-dist:** The executor must run `npm install pdfjs-dist` before creating vendorSync.js to ensure the import resolves.
